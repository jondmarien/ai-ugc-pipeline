Copy Fail (CVE-2026-31431) lets an unprivileged user write four controlled bytes into the page cache of any readable file. A 732-byte Python script edits a setuid binary and gains root.

Disclosed April 29 by Xint.io and Theori. CVSS 7.8. The exploit is portable, tiny, stealthy, and cross-container — four properties that almost never appear together. Works on every Linux distribution shipped since 2017.

The flaw is in the algif_aead crypto module. An in-place optimization from August 2017 let a page-cache page end up in the kernel writable scatterlist for an AEAD operation over AF_ALG socket. Unprivileged process drives splice() into that socket and completes a targeted write into the page cache.

Four steps: open AF_ALG socket, construct payload, trigger write to cached /usr/bin/su, execve the poisoned su as root. No race condition. No kernel offsets. Same exploit across distros.

Mitigation: blocklist algif_aead module. But Dirty Frag and Fragnesia bypass this entirely using xfrm and RxRPC paths. The bug class is wider than one module.

Every major distro released advisories. CVE-2026-31431 patched in mainline. Dirty Pipe (2022) was the same class in a different subsystem.

The 2017 optimization broke the page-cache isolation guarantee. Nine years later, a 732-byte script exploits it on every distro.
